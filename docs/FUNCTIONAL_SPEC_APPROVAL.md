## 1. วัตถุประสงค์ของเอกสาร (Purpose)

เอกสารฉบับนี้จัดทำขึ้นเพื่อระบุขอบเขตฟีเจอร์และความสามารถของระบบ **Payment Backoffice System** ที่ได้รับการพัฒนาขึ้น เพื่อให้ผู้มีส่วนเกี่ยวข้องรับทราบ ตรวจสอบ และลงนามยืนยันความถูกต้องก่อนนำระบบไปใช้งานจริง

This document defines the scope of features and capabilities of the **Payment Backoffice System** for stakeholder review and formal sign-off prior to production use.

---

## 2. ภาพรวมระบบ (System Overview)

ระบบ Payment Backoffice เป็นแอปพลิเคชันสำหรับเจ้าหน้าที่การเงินโรงเรียน ใช้สำหรับบริหารจัดการค่าธรรมเนียมการศึกษา กิจกรรมนักเรียน การออกใบแจ้งหนี้ การชำระเงิน และการออกใบเสร็จรับเงิน รองรับการทำงานหลายบทบาท (Role-based Access Control)

**บทบาทผู้ใช้งาน (User Roles)**

| บทบาท | สิทธิ์การใช้งาน |
|-------|----------------|
| Admin | เข้าถึงได้ทุกฟีเจอร์ |
| Admin Accountant | สร้าง / จัดการ / อนุมัติใบแจ้งหนี้ และจัดการส่วนลด |
| Viewer | ดูข้อมูลได้อย่างเดียว ไม่สามารถแก้ไขได้ |
| Approver | เข้าถึงเฉพาะคิวอนุมัติใบแจ้งหนี้ |

## 3. รายการฟีเจอร์ที่ระบบรองรับ (Feature List)

> **คำอธิบายสัญลักษณ์**
> - ✅ = มีฟีเจอร์นี้
> - — = ไม่มีฟีเจอร์นี้ในหน้าดังกล่าว
> - **CRUD** = เพิ่ม (Create) / ดู (Read) / แก้ไข (Update) / ลบ (Delete)
> - **Import** = นำเข้าข้อมูลจากไฟล์ Excel/CSV
> - **Export** = ส่งออกข้อมูลเป็นไฟล์ (Excel/PDF/ZIP)
> - **Filter** = กรอง/ค้นหาข้อมูล
> - **Pagination** = แบ่งหน้าข้อมูล
> - **Sort** = เรียงลำดับข้อมูล
> - **Bulk** = ทำงานกับหลายรายการพร้อมกัน
> - **PDF** = สร้าง/ดาวน์โหลด PDF
> - **Email** = ส่งอีเมลจากระบบ
